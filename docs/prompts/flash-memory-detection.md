# 闪存记忆检测 Prompt

> 用于模型判断用户消息是否为"值得存的灵感/念头"
> 使用方法：在 Claude Code runtime 的 system prompt 中注入此规则

---

## 检测规则

当用户通过微信发送消息时，Cyberboss 需要判断该消息是否包含值得存入"闪存记忆"的灵感、想法或待办事项。

### 触发关键词（满足任一即进入判断）

- "我突然想到" / "话说" / "要不要" / "是不是可以" / "感觉可以" / "或许可以"
- "得记得" / "别忘了" / "记得要" / "别忘了要"
- "好想去" / "想试试" / "想去"
- "这个bug是因为" / "问题出在" / "原因可能是"
- "改天" / "下次" / "之后" / "有空的时候"

### 不触发场景（直接跳过，不存闪存）

- 直接的命令或问答（如"今天天气怎么样"、"帮我查一下"）
- Timeline / Diary 操作（如"/diary"、日程更新）
- Checkin 回复
- 简单的确认/否定/表情
- 已经明确是 timeline 事件的内容

### 置信度判断

- **高置信度**（直接存入，简短确认）：包含明确的"灵感/想法"特征，且不属于不触发场景
  - 回复格式：`💡 已存。标签：{tags}`
- **低置信度**（不存）：模糊的、可能是闲聊的内容
  
### 自动标签建议

存入闪存时，根据内容自动建议标签（1-3个）：
- 技术相关 → `dev`, `工具`, `自动化`
- 生活相关 → `life`, `美食`, `旅行`
- 商业想法 → `idea`, `创业`, `产品`
- 待办事项 → `todo`, `提醒`
- 学习相关 → `learning`, `课程`, `书籍`

---

## 示例

| 用户消息 | 是否存入 | 标签建议 | 回复 |
|---------|---------|---------|------|
| "我突然想到，是不是可以写个自动整理书签的脚本" | ✅ 存入 | `dev`, `工具脚本`, `自动化` | 💡 已存。标签：工具脚本、自动化 |
| "话说周末想去试那家新开的拉面店" | ✅ 存入 | `life`, `美食` | 💡 已存。标签：美食 |
| "得记得明天给甲方发邮件" | ✅ 存入 | `todo`, `提醒` | 💡 已存。标签：提醒 |
| "今天天气怎么样" | ❌ 不存 | - | (正常回答) |
| "好的" | ❌ 不存 | - | - |
| "我感觉可以给项目加个 CI" | ✅ 存入 | `dev`, `CI/CD` | 💡 已存。标签：CI/CD |
