1. 版本管理

  • 版本管理策略同意执行。

  • 项目名称：customized_cyberboss_xiaofang。

2. 碎片化记忆

  • 题目和解析要出现在每日总结上。
  • 知识库初始内容：由我来整理，请告诉我我所需要的提供的格式，以便后续增加知识库。
  • 知识库补充：根据用户历史日程内容来更新，在用户工作室后台进行搜索整理。
  • 构建新的知识库时，在每日总结处可以说自己搜寻构建了哪些知识库，存放在 Obsidian 仓库哪里（如：概率论统计——关于怎么精准）。
  • 追问机制：在用户随口构建日程时，Cyberboss 可以进行追问，并且按照模板分类整理。例如用户说“要学概率论统计”，可以追问：哪本书？听谁的课？哪一章的什么内容？

3. 每日总结 （#和上下文与对话框管理有关项）

  • 每日总结放在 PC 的前端网站中，同样以网页截图的形式发给用户；同时在 Obsidian 存放一版 .md 格式。
  • 分级：日/月/年总结。
  • 触发形式采用混合模式：
    • 定时每日 21:00 生成今日总结草稿。
    • 用户确认今天收工，补充内容，放出完整版（若用户在 21:00 前收工就立即开始生成）。
  • 模板：
    • .md 格式暂时使用你提到的模板。
    • 网页 HTML 格式的模板要再更改给我看看（依照原项目的基础改）。
  • 每日总结的思考 prompt 大致结构：待补充。

4. 闪存记忆

  • 需单开一个 Obsidian 仓库隔离整理。
  • 整理逻辑：去重、合并、关联算法需要怎么设计？
  • 我想使用调用 API 进行操作，在每日总结时去仓库调取闪存胶囊的内容，使用 AI 进行去重关联。
  • prompt 大致方向：待补充。

5. 大构思完善 （#增加版本管理思维，甚至是类似 git graph 插件的可视化思维）

  • 碎片化记忆和该功能采取同一“空闲判定机制”：
    • 日程更新 → 重新计算闲暇时间 → 重新产生闲暇随机数（偏向闲暇时间的中旬）。
    • 如果闲暇时间 < 30min → 进行碎片记忆。
    • 如果闲暇时间 > 30min → 进行大构思完善。
  • 在更新日程后，把提问内容和提问具体时间也要标入日程。
  • 在开始前一定要询问，如果用户现在不想就取消。
  • 每日开机时刷新读取。
  • 用户在 Obsidian 提交大构思后，AI 要针对大构思进行一些合理化和可操作行为完善此 .md 文档，并且要生成一个问题表格，问过的自动打上勾并梳理用户回答，对之后的问题也提供动态调整 （#和上下文与对话框管理有关项）。
  • 提问引擎的 prompt：待补充。
